## 第二十章 Hermes vs GenericAgent 到底差在哪

> 🎙️ **老师总结**：好了，全部核心模块都讲完了。现在是收尾时刻——把你之前学的 GA 和今天学的 Hermes 放在一张桌子上，正面 PK。这不是比谁好谁差，而是让你看到**同一个问题的两种解法**。GA 追求的是"极简——给 Agent 最少的预设能力，让它自己长"，Hermes 追求的是"全家桶——开箱就有 40+ 工具、20+ 平台、18+ 模型"。GA 像裸机 Linux，Hermes 像 macOS。这套对比你面试一定能用上，因为面试官最爱问的就是"你用过 XX 吗，跟 YY 比怎么样？"

### 20.1 全面对比表

| 维度 | **GenericAgent** | **Hermes Agent** |
|:---|:---|:---|
| **代码量** | ~3K 行 | ~35K 行 |
| **设计哲学** | 极简主义，不预设技能 | 工业级，内置学习闭环 |
| **核心工具** | 9 个原子工具 | 40+ 内置工具 |
| **工具策略** | 靠 code_run 动态创建 | 内置常用 + 动态扩展 |
| **记忆** | L0-L4 五层文件系统 | 两文件 + 全文搜索 + 用户画像 |
| **记忆容量** | 动态 | ~3,600 字符（故意限制） |
| **技能系统** | 自动固化为 Skill | 闭环自进化（创建→使用→修复） |
| **技能修复** | 无主动修复 | 发现问题立即 patch |
| **上下文压缩** | 有（L4 会话归档） | 四阶段压缩流水线 |
| **模型** | ~5 家 | 18+ 家，200+ 模型 |
| **模型路由** | 无 | 智能路由（省 30-50%） |
| **执行后端** | 本地为主 | 6 种 |
| **消息平台** | 6 个 | 20+ 个 |
| **MCP** | 无原生支持 | Client + Server |
| **RL 训练** | 无 | 数据飞轮 + Atropos |
| **子 Agent** | Conductor 编排 | delegate_task（严格约束） |
| **安全** | ask_user 确认 | 多层防护（12 种注入检测 + ...） |
| **并行工具** | 无 | 白名单制（最多 8 线程） |
| **License** | MIT | MIT |

### 20.2 各自的优势

**GA 的优势**：
- 代码量小，容易读懂
- Token 效率极高（<30K 上下文窗口）
- 9 个原子工具设计很优雅
- 适合学原理
- 自举实证（项目自身由 Agent 创建）

**Hermes 的优势**：
- 工程完整度高，开箱即用
- 模型兼容性极强
- 平台覆盖广
- 安全体系完善
- RL 训练数据飞轮（独有）
- 云原生执行后端
- MCP 原生支持
- 开放标准技能

### 20.3 什么时候用哪个

| 场景 | 推荐 | 原因 |
|:---|:---|:---|
| 学 Agent 原理 | GA | 代码精简，好读 |
| 快速原型 | GA | 轻量，启动快 |
| 长期个人助手 | Hermes | 闭环学习，越用越强 |
| 企业级部署 | Hermes | 安全完善，多后端 |
| 多平台机器人 | Hermes | 20+ 平台 |
| RL 训练研究 | Hermes | 独有数据飞轮 |
| 极致省 Token | GA | <30K 上下文 |
| 需要 MCP | Hermes | 原生支持 |

### 20.4 一句话总结

> **GA 是内功心法（精简、核心、透彻），Hermes 是外功招式（全面、实用、工业级）。两者都练，内外兼修。**

## 第二十一章 学完之后怎么跟别人讲

### 21.1 三分钟讲完 Hermes Agent

> "Hermes Agent 是 Nous Research 做的开源 AI 智能体框架，核心特点是'越用越聪明'。
>
> 它的架构是四层楼：
> - **第一层——记忆**：两个文件（MEMORY.md 存事实，USER.md 存用户画像），加一个全文搜索引擎翻历史聊天。
> - **第二层——技能**：把'怎么做'存成 SKILL.md 文件，需要时才加载（省 Token）。
> - **第三层——自进化**：Agent 做完复杂任务自动存技能，用技能时发现过时了自动修复，五步闭环全自动。
> - **第四层——上下文压缩**：超长对话时自动压缩，头尾保护中间摘要，技能和记忆都在文件里不怕丢。
>
> 工程上：支持 18+ 家模型、20+ 消息平台、6 种执行后端、40+ 内置工具、MCP 原生支持。
>
> 最独特的是 RL 训练系统——Agent 执行任务生成轨迹数据，喂回去训练模型，形成正反馈循环。这是其他 Agent 框架都没有的。
>
> 和 GenericAgent 比：GA 是极简种子代码（3K 行），Hermes 是工业级全家桶（35K 行）。GA 适合学原理，Hermes 适合实际用。"

### 21.3 学习建议

1. **带着对比学**：每学一个 Hermes 模块，想想"GA 里怎么做的？为什么 Hermes 选了不同的实现？"

2. **重点理解四层能力**：第四章是总纲，理解透了后面就是在展开。

3. **动手实验**：装上 Hermes，实际用一段时间，观察它怎么记忆、怎么创建技能、怎么自我修复。

4. **读源码**：从 `run_agent.py` 的 `run_conversation()` 开始，跟着调用链走。

5. **关注安全设计**：Hermes 的安全体系是 GA 没有的，值得研究。

6. **RL 训练是加分项**：对模型训练感兴趣的，数据飞轮值得深入研究。
