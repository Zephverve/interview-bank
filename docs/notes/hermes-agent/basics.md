## 第一章 这玩意儿到底是个啥

### 1.1 名字怎么来的

"Hermes"这个词三个意思：
1. 希腊神话里的**信使之神**——跑得快，消息灵
2. 奢侈品品牌**爱马仕**（Hermès）——贵，而且越用越有包浆
3. Nous Research 出的**AI 模型和 Agent 框架**——你今天要学的

中国社区管它叫"爱马仕 Agent"，是因为它有个特点跟爱马仕包包一样：**越用越值钱**。第一天用可能觉得也就那样，用了一百天之后，它已经记住你的习惯、学会你的工作方式、修好了自己的 bug——完全不是第一天那个水平了。

### 1.2 一句话定义

> **Hermes Agent 是一个开源的 AI 智能体框架，最大的特点是"会用越聪明"——它能记住你说过的话、自动学会怎么干活、在使用中自己修自己的 bug，全程不需要人去微调模型。**

### 1.3 和你已经学的 GenericAgent 有啥不一样

你已经学过 GenericAgent（以下简称 GA），那咱们直接上对比，不废话：

| 你关心的点 | GenericAgent（你已学） | Hermes Agent（你要学的） |
|:---|:---|:---|
| **代码量** | ~3000 行，精简到骨头 | ~35000 行，工业级全家桶 |
| **工具数量** | 9 个原子工具 | 40+ 内置工具 |
| **记忆** | L0-L4 五层文件系统 | 两个文件 + 全文搜索 + 用户画像 |
| **自我进化** | 任务完成自动固化技能 | 闭环：创建→使用→发现问题→修复 |
| **模型** | Claude/Gemini/Kimi 等 ~5 家 | 18+ 家，200+ 模型 |
| **消息平台** | 6 个 | 20+ 个 |
| **代码在哪跑** | 基本在本地 | 6 种地方（本地/Docker/云/...） |
| **核心区别** | 极简种子，靠生长 | 工业级成品，内置学习闭环 |
| **RL 训练** | 没有 | 有（这是 GA 完全没有的） |

**打个比方**：GA 像是一辆手动挡小跑车——零件少、好拆、好研究，适合学原理；Hermes 像是一辆自动挡 SUV——功能多、开箱即用、还能自己学习你的驾驶习惯。

## 第二章 谁造的，什么来头

### 2.1 Nous Research 是什么组织

不是什么大厂部门。2023 年，一群 AI 爱好者在 Discord 社区里搞开源协作，慢慢搞成了一个正式的 AI 研究实验室。

**灵魂人物**：Teknium，Hermes 系列模型的创造者。

**融资**：拿了 5000 万美元 A 轮，投资方是 Paradigm 和 North Island Ventures——都是加密货币领域的大佬，所以融资方式也带着 Web3 味道（代币计价而非传统股权）。

**使命**：通过开源语言模型推进人权和自由。这个使命决定了 Hermes 的"无审查/最少过滤"风格。

### 2.2 他们不只是做 Agent

Nous Research 是**既造模型又造框架**的：

| 产品 | 干什么的 |
|:---|:---|
| Hermes 模型系列 | 旗舰开源大模型，已出到 Hermes 4 |
| **Hermes Agent** | AI 智能体框架 ← 你学的这个 |
| Nous Portal | API 服务，接 400+ 模型 |
| Psyche | 去中心化分布式训练网络 |
| NousCoder | 代码生成模型 |

### 2.3 模型进化史（简单了解就行）

| 时间 | 模型 | 关键进步 |
|:---|:---|:---|
| 2023 初 | Hermes 原版 | 能听懂指令 |
| 2024.03 | Hermes 2 Pro | 支持函数调用（Function Calling） |
| 2024.08 | Hermes 3 | 旗舰版，有 Agent 能力了 |
| 2025.06 | Hermes 4 | 推理和对话统一了 |

**重点**：他们是做模型出身的人，所以 Hermes Agent 里有一个其他 Agent 框架都没有的东西——**RL 训练系统**（后面第十七章细讲）。这意味着 Agent 不光能在运行时学（记技能），还能把学到的经验喂回去训练模型本身。

## 第三章 一句话讲清核心思想

### 3.1 普通 Agent 的毛病

你想想你已经熟悉的 GA 和其他 Agent 有什么通病：

```
毛病1：问完即忘。你今天告诉它"我用 pnpm 不用 npm"，明天它又问你用什么包管理器。
毛病2：不学新东西。配好了什么能力就只有什么能力，不能自己长本事。
毛病3：靠人维护。技能需要人写、人装、人更新，Agent 自己不会。
毛病4：会话隔离。每次对话都是独立的，上次的经验用不上。
```

GA 部分解决了这些问题（任务完成自动固化技能）。但 Hermes 把这个理念推到了更狠的程度。

### 3.2 Hermes 的核心思想（就一句话）

> **Agent 应该从和你的交互中学习，越用越强，不需要微调模型，不需要人手动维护。**

翻译成大白话：**不是"你用得更好"，而是"越用越好"。**

普通 Agent：第一天能用，第一百天还是那个水平。
Hermes Agent：第一天能用，第一百天比第一天强——因为在这 100 天里它干了四件事：
1. **记住**了你的偏好和习惯
2. **学会**了怎么完成各类任务
3. **修好**了过时的技能
4. **没忘**之前的工作（超长对话也不丢信息）

**再打个比方**：GA 像一个聪明的实习生——每次都能完成任务，但你得反复告诉他"我们用 pnpm"。Hermes 像一个会做笔记的老员工——告诉他一次，他就记住了，下次不会再犯。
