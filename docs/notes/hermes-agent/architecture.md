## 第四章 整体架构——四层楼怎么盖起来的

> 🎙️ **老师开讲**：前导课里我给你看了那栋四层楼的全景图——一楼记忆、二楼技能、三楼自进化、四楼持久。现在咱们走近这栋楼，一层一层看里面到底装的什么。记住一个关键：这四层不是"功能 A + 功能 B"的并列关系，是一层一层往上盖的——**每一层都是为了补上一层的缺口**。先看一楼解决了什么、还缺什么，再看二楼怎么补这个缺口，以此类推。这就像盖真的房子——你不会先盖三楼再回来打地基。来，咱们进门。

这一章是整个 Hermes 的**总纲**。理解了这四层，后面所有章节都是在展开每一层的细节。

### 4.1 不是并列功能，是递进盖楼

Hermes 的设计可以拆成**四层能力**。注意，这四层不是"功能 A + 功能 B"的并列关系，而是一层一层往上盖的——**每一层都是为了补上一层的缺口**。

给你画个图：

```
第一层：能记住  —— 解决"问完即忘"
    │
    │  缺口：记住了"用户用 pnpm"这个事实，
    │  但记不住"怎么部署到 Vercel"这个操作流程
    ▼
第二层：能复用  —— 解决"每次从头摸索"
    │
    │  缺口：技能需要人手动创建和维护，
    │  Agent 自己不会学也不会修
    ▼
第三层：能进化  —— 解决"Agent 自己学自己修"
    │
    │  缺口：超长对话时上下文窗口溢出，
    │  前面创建的技能和经验全丢了
    ▼
第四层：能持久  —— 解决"长对话不丢进化链"
```

### 4.2 四层总结表

| 第几层 | 能力 | 解决什么问题 | 上一层的缺口是什么 |
|:---:|:---|:---|:---|
| ① | **能记住** | 跨会话保持事实和偏好 | ——（这是起点） |
| ② | **能复用** | 编码和复用方法论 | 记得住事实，但记不住操作流程 |
| ③ | **能进化** | 自主创建和修复技能 | 技能需要人创建维护 |
| ④ | **能持久** | 长对话中不丢进化成果 | 上下文溢出，前面的工作丢失 |

### 4.3 如果你跟别人解释

> "Hermes Agent 的架构就是四层楼：第一层让它能记住事，第二层让它能记住怎么干活，第三层让它能自己学自己修，第四层保证超长对话也不丢东西。每一层都是在补上一层的漏洞。"

接下来的第五到第八章，就是把这四层一层一层掰开讲。
