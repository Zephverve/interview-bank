### 0.1 一个故事：两个 AI 助手的差距

想象你有两个助手：小普和小马。

**小普是个普通 AI 助手。** 第一天你告诉它"我的项目用 pnpm，别用 npm"，它说好。第二天你让它装依赖，它上来就是 `npm install`。你吼它——"我昨天不是说了用 pnpm 吗！"它一脸无辜："啊？昨天？昨天的对话已经清空了啊……"

**小马呢？它是 Hermes Agent。** 第一天你说"用 pnpm"，它掏出一个小本本记下来——"此人用 pnpm"。第二天你让它装依赖，它翻开小本本一看——哦，用 `pnpm install`。不用你吼。

这只是开始。一个月后，你让小马部署项目，它在任务中发现了规律——"这个部署流程我已经做了 6 次了，每次都是同样的步骤"。于是它把"如何部署"写成了一个小教程，贴在墙上。下次你说"部署"，它直接按教程走，不用每次重新摸索。

两个月后，Vercel 改了命令——旧的 `vercel deploy` 不管用了。小马在按教程执行时发现了这个问题……然后**它自己把教程改了**。你没注意到，你只知道"部署成功了"。

这就是 Hermes Agent 的核心能力——**越用越聪明，不需要你微调模型，不需要你手动维护**。

### 0.2 一张图看懂 Hermes 怎么工作的

别慌，这张图现在很多词你可能不认识，没关系——先有个印象就行：

```
你发一句话（随便在哪——飞书、微信、命令行都行）
          │
          ▼
    ┌─────────────┐
    │  Hermes     │  ← 一个总管进程，接收来自各平台的消息
    │  Gateway    │
    └──────┬──────┘
           │
           ▼
    ┌──────────────────────────────────────┐
    │          Agent 主循环（大脑）         │
    │                                      │
    │   ┌── 先加载记忆 ──────────────┐     │
    │   │  "此人用pnpm""项目在Vercel上"│     │

    │   └───────────────────────────┘     │
    │                                      │
    │   ┌── 再看有没有能用的技能 ────┐     │
    │   │  "部署到Vercel"的技能？有！  │     │
    │   │  直接加载教程按步骤走       │     │
    │   └───────────────────────────┘     │
    │                                      │
    │   ┌── 调 AI 大模型推理 ───────┐     │
    │   │  "这个任务需要什么工具？"   │     │
    │   │  可以选 GPT/Claude/DeepSeek │     │

    │   │  共 40+ 种内置工具         │     │
    │   └───────────────────────────┘     │
    │                                      │
    │   ┌── 任务完成后 ────────────┐     │
    │   │  "这个流程值得存下来吗？"  │     │
    │   │  值得 → 自动写教程        │     │
    │   │  教程过时了 → 自动修      │     │
    │   └───────────────────────────┘     │
    │                                      │
    └──────────────────────────────────────┘
```

**看到没，核心就四个字：循环 + 进化。**

Agent 在转圈圈——收到消息 → 看记忆 → 看技能 → 调用 AI → 执行工具 → 看结果 → 再调用 AI……直到任务完成。而在这个过程里，它会自己总结规律、自己存技能、自己修教程。

### 0.3 四层楼——整个 Hermes 的骨架

你记住一个比喻就够了：Hermes 是**一栋四层楼**。每一层盖在上一层上面，补上一层的缺口：

```
第四层：能持久 ← 对话太长前面的东西会不会丢？不会，压缩技术保护一切
  ▲
第三层：能进化 ← Agent 自己能学会新技能，发现技能过时了能自己修
  ▲
第二层：能复用 ← 复杂流程写成"菜谱"（SKILL.md），下次按菜谱走
  ▲
第一层：能记住 ← 记住关于你的事实和偏好（MEMORY.md + USER.md）
```

| 楼层 | 类比 | 解决了什么 | 还缺什么 |
|:---|:---|:---|:---|
| **一楼 记忆** | 冰箱上的便签条 | 问完即忘 | 便签写不下"怎么部署到 Vercel"这种完整流程 |
| **二楼 技能** | 墙上贴的菜谱 | 每次从头摸索 | 菜谱会过时（API 改了、命令变了），需要人维护 |
| **三楼 自进化** | 厨师自己写菜谱自己改 | 技能要人维护 | 对话太长前面的工作可能被挤出上下文 |
| **四楼 持久** | 防火保险柜 | 上下文溢出丢失成果 | ——（四条全齐了） |

**记住这个四层楼，后面的每一章都是其中一层的展开。** 第五章讲记忆（一楼），第六章讲技能（二楼），第七章讲自进化（三楼），第八章讲持久（四楼）。

### 0.4 跟 GenericAgent（你已经会的）有什么本质区别

你学过 GenericAgent（GA），那咱们直接上对比，不废话：

| 你关心的 | GA（内功心法） | Hermes（外功招式） |
|:---|:---|:---|
| **代码量** | ~3000 行，你花一个下午能看完 | ~35000 行，工业级 |
| **核心哲学** | 极简——不预设技能，让 Agent 自己长 | 全家桶——内置 40+ 工具，开箱即用 |
| **自我进化** | 任务完成后自动存技能 | 闭环——创建→使用→发现问题→自动修复 |
| **模型** | 5 家左右 | 18+ 家，一句话切换 |
| **消息平台** | 6 个 | 20+ 个，从飞书到 WhatsApp 全通 |
| **RL 训练** | 没有 | **独有！** Agent 执行任务的数据能喂回去训练模型 |

一句话总结：**GA 教你 Agent 怎么"动"，Hermes 教你 Agent 怎么"活"。**

### 0.5 前导课小结——这几句话你先记住

看完后面的细节如果有点晕，回来翻这一页就行：

> ① Hermes Agent 是一个开源的 AI 智能体，最大特点是**越用越聪明**——它不是装好就定型的工具，而是一个会成长的助手。
>
> ② 核心机制是**自进化闭环**——Agent 做完复杂任务后自己写教程（技能），下次直接用；发现教程过时了自己改。
>
> ③ 架构是**四层楼**：先能记住（记忆），再能复用（技能），然后能自我进化（闭环），最后保证长时间不丢（压缩）。
>
> ④ 跟 GA 比，GA 是"极简种子"，Hermes 是"工业级全家桶"。GA 适合学原理，Hermes 适合实际用。
